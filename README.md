## Sobre o Projeto

O **PortfolioHUB** é uma plataforma digital projetada para centralizar minha trajetória acadêmica e profissional. O objetivo é demonstrar habilidades práticas em engenharia de software, apresentando minha transição de carreira para a área de TI. O site consolida meu currículo, competências técnicas e os principais softwares desenvolvidos.

### Tecnologias Utilizadas no Frontend
* **HTML5:** Estruturação semântica do conteúdo.
* **CSS3:** Estilização, responsividade (Mobile First) e animações.
* **JavaScript:** Interatividade e consumo dinâmico de dados.
* **Vercel:** Hospedagem em nuvem e pipeline de CI/CD.

## 📂 Projetos em Destaque no Hub

Este repositório serve como ponto de partida para os meus principais trabalhos práticos:

1. **[Sistema de Gestão de Custos de Obras]** - Plataforma backend desenvolvida em Node.js e PostgreSQL para gerenciar materiais, mão de obra e notas fiscais em canteiros de obra.
2. **[Dom Imperial (WaaS)]** - Protótipo funcional de Website as a Service desenvolvido para o comércio local, focado em geração de receita recorrente.
3. **[Biblioteca Digital Python]** - Sistema de catalogação de livros estruturado com Programação Orientada a Objetos e manipulação de tuplas em Python.
4. **[Análise de Dados Físicos]** - Estudo de conversão de energia via simulador PhET com processamento de dados e geração de gráficos utilizando a biblioteca Pandas (Python).

## ⚙️ Arquitetura, Segurança e Implantação

Para atender aos rigorosos critérios de controle de qualidade e implantação profissional, este repositório conta com documentações específicas:

* 📄 **[DEPLOYMENT.md](./DEPLOYMENT.md):** Detalha a arquitetura de implantação na Vercel, configuração de DNS para o domínio customizado e a integração contínua (CI/CD) atrelada à branch `main`.
* 🛡️ **[SECURITY.md](./SECURITY.md):** Estabelece as políticas de segurança do projeto, incluindo regras de proteção de branch (Branch Protection Rules), ocultação de variáveis de ambiente e a automação de varredura de vulnerabilidades através do GitHub Dependabot.

---

## 🤝 Workflow de Desenvolvimento (Colaboração)

O repositório adota um fluxo de trabalho estruturado e seguro:
1. A branch `main` é **protegida** contra commits diretos.
2. Novas funcionalidades ou inclusões de projetos devem ser feitas em ramificações isoladas (ex: `feature/novo-projeto`).
3. A integração de código ao ambiente de produção exige a abertura de um **Pull Request (PR)** e validação estática.
